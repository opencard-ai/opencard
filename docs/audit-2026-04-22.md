## 執行摘要 (TL;DR)

上次的 schema 建議**部分採納**:卡片去重從 254 → 249、`insurance` 型別收斂成 object、`earning_rates` shape 從 11+ 種降到 4 種、`tags` 收斂成 11 個標籤、`last_updated` 全部用 ISO datetime。**但還有 6 個關鍵問題沒修掉**,其中最嚴重的兩個是:

1. **`welcome_offer` shape 從 13 種增加到 20 種** — 不減反增,表示新資料仍是自由填寫。
2. **`recurring_credits` 涵蓋率只從 25% → 30%(75/249)** — 這個欄位是網站主打功能「每月福利提醒」的資料來源,只有 3 成卡片有資料,電子報對大多數使用者會空白。

除了資料層,**可用性和無障礙層有四個立刻可改的大洞**:

- 首頁 DOM 尺寸 127,768 px(所有 249 張卡一次渲染完) — 手機滾動體驗極差,SEO 訊號稀釋。
- 首頁 508 顆按鈕中有 498 顆只是 emoji(⚖/💾),**沒有 aria-label** — 螢幕報讀器使用者完全無法操作。
- 8 個表單控制項(搜尋框、語言、發卡行、類型、年費、排序濾鏡)**全部沒有 label**。
- 首頁有 252 個 `<h2>`(每張卡是一個 h2)— 標題階層等於沒有階層。
- 卡片詳細頁**沒有 JSON-LD 結構化資料、沒有 og:image、沒有 canonical**,249 個 /cards/* 路徑**沒有列入 sitemap.xml**,這是非常顯眼的 SEO 自損。

## 3. 網站架構 (Technical Architecture)

### 效能問題
- 首頁 HTML decoded = 551 KB,DOM nodes = **6,536 個**
- **首頁 scrollHeight = 127,768 px**(≈ 160 個螢幕高度)
- 沒有分頁/懶加載/虛擬列表

### SEO 問題
- `sitemap.xml` 只有 6 條:249 個卡片詳細頁**全部沒進 sitemap**
- 每個卡片詳細頁:**���有 `<link rel="canonical">`、沒有 `<meta property="og:image">`、沒有 `<script type="application/ld+json">`**
- 首頁有 1 個 `<h1>` + **252 個 `<h2>`**

### 安全 headers
- 唯一的安全 header: `Strict-Transport-Security: max-age=63072000`
- 缺少: CSP, X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy

## 4. 使用體驗 (UX)

### 首次訪客問題
- 主要 CTA 有 4~5 個,視覺等權重,決策疲勞
- 首頁沒有 hero-level 教學說明網站定位
- AI 互動性被藏在 query string

### 核心功能問題
- 只有 30%(75/249)的卡有 `recurring_credits` 資料 — **這是「每月福利提醒」最嚴重的空白**

### 卡片詳細頁問題
- 沒有 Compare 按鈕、沒有 Share / Copy Link、沒有 Print、沒有 breadcrumb
- **沒有明顯的「Track this card」CTA**
